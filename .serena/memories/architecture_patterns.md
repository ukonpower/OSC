# OREngine アーキテクチャパターンと設計ガイドライン

## コンポーネントベースアーキテクチャ
OREngineはコンポーネントベースのアーキテクチャを採用しており、以下の要素で構成される：

### エンティティシステム
- Entity: シーングラフの基本単位
- Transform: 位置、回転、スケール管理
- 階層構造による親子関係管理

### 標準コンポーネント
- Mesh: メッシュレンダリング
- Camera: カメラ制御
- Light: ライティング
- カスタムコンポーネント対応

### レンダリングパイプライン
- Deferredレンダリング
- PBRマテリアル
- ポストプロセス
- カスタムシェーダー

## パッケージ構成
- `orengine/`: コアエンジン機能
- `glpower/`: WebGLユーティリティ (サブモジュール)
- `maxpower/`: 最適化ユーティリティ

## Reactインテグレーション
- JSX/TSXコンポーネントとの統合
- React HooksでのEngine状態管理
- Storybookでのコンポーネントテスト

## Blender連携
- BLidgeシステムによるアセット連携
- データ変換パイプライン
- シーンエディター統合

## 設計原則
- 64KB制限を意識したコードサイズ最適化
- モジュール性とスケーラビリティ
- 型安全性の重視
- 高速な開発体験の提供
